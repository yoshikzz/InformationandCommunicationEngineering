# 2.0 情報通信の基礎
本章では、情報通信に使われる各種伝送路を概観し、通信の妨げとなるいくつかの要因について触れる。
そのあと、情報伝送の基礎である符号化について、情報源符号化、通信路符号化、伝送路符号化に分けてその考え方を簡単に説明し、それぞれの役割について理解を深めてもらう。最後に、情報ネットワークで重要となる情報セキュリティについて軽くまとめる。

## 2.1 情報伝送路
我々の身の回りを眺めてみると、種々の情報伝送路が使われていることに気づく。情報伝送路とは、情報を一点から別の点へ届けるための道のことである。大きくは優先と無線に分類され、用途・距離・経済性などに応じて使い分けられる。
まず有線伝送路を見ていこう。家庭には電話用の**より対線**（ツイストペアケーブル）が引き込まれており、それを用いて世界中の人々と即時自動通話が可能になっている。また、ファクシミリやパソコンも同じ回線に接続できる。CATVのサービス地域では、**同軸ケーブル**あるいは**光ファイバケーブル**によって各種映像情報が配信されている。ビル内のオフィスに目を向けると、多数の多機能電話機やデジタルPBX（private branch exchange：構内交換機）が電話線によって接続されているほか、各種情報処理機器が同軸ケーブルや光ファイバケーブルを用いたLAN（ローカルエリアネットワーク）によって結合されており、さらに外部の広域ネットワークともつながっている。

次に無線伝送路である。もっとも身近なものはテレビ・ラジオ放送であり、静止衛星を使った衛星放送も一般的になってきた。電話局の鉄塔に目を向ければ、マイクロ波中継伝送用のホーンアンテナやパラボラアンテナが見える。また、家庭ではコードレス電話、外出中は自動車電話・携帯電話と、無線伝送路は私たちの日常生活に深く浸透している。カーナビゲーションシステムの実用化も近づいており、電波を効率的に利用した新しい**移動通信方式**が導入されつつある。

| 伝送路の種類 | 主な用途         | 損失の特性           | 中継器間隔   |
| ------ | ------------ | --------------- | ------- |
| より対線   | 家庭への電話引込線    | 周波数の平方根に比例      | 数km     |
| 同軸ケーブル | 長距離多重伝送・CATV | 周波数の平方根に比例      | 数km     |
| 光ファイバ  | 幹線・海底ケーブル    | 非常に小さく周波数に依存しない | 数10km以上 |
| 電波（無線） | 移動通信・放送・衛星   | 距離の二乗に比例（自由空間）  | —       |
それでは、これらの伝送路はどのような理由で選択されるのだろうか。主な判断基準は、送りたい情報の量・距離・信号の特性・敷設条件・経済性などである。たとえば、最寄りの電話局から各家庭まで数kmの区間は電話信号のみを伝送すればよく、より対線がもっとも経済的な選択肢となる。一方、多数の電話信号を長距離にわたって伝送する場合には、1本の高品質なケーブルに多数の信号をまとめて乗せる**多重化**が有効である。高価なケーブルを1本敷設しても、多重化によって多くの通信を同時に流せるため、1通信あたりのコストを下げることができる。

従来この多重化には同軸ケーブルが用いられてきたが、光ファイバケーブルが安価に生産できるようになるにつれ、現在ではたった2本（往復各1本）の光ファイバで**20,000人分**の電話信号を伝送できるまでになった。各家庭への引込線については、多重化が難しいため安価な伝送路（より対線）が使われ続けてきた。しかしこれからは、動画像情報など広帯域の信号伝送への需要が高まることが予想されており、光ファイバケーブルへの置き換えが進む傾向にある。

なお、デジタル信号伝送方式の代表例である**PCM（パルス符号変調）通信**は、当初は長距離伝送に向いた方式とみなされていたが、近距離の悪い回線でも利用できる点が注目され、最初の近距離実用化は日本では1965年であった。

最後に、伝送路の損失について整理しよう。銅線などのメタルを使った伝送路は損失が大きく、その損失は伝送周波数の平方根に比例して増大する。このため、数kmごとに中継器を挿入して信号を増幅・再生する必要があった。一方、光ファイバケーブルは損失がきわめて小さく、しかも損失が伝送周波数にほとんど依存しない。このため、数10km以上の間隔で中継器を設置すればよく、長距離伝送や国際海底ケーブル伝送にとってきわめて魅力的な伝送路となっている。

## 2.2 通信を妨げる要因 -雑音・干渉・ひずみ
通信の目的は、伝えたい情報をいかにして誤りなく空間的距離を隔てた遠方の相手方に伝えられるかにある。しかしながら、通信を行おうとするとき、送りたい信号以外に、多かれ少なかれ邪魔者の信号が入ってくる。希望信号以外の不規則性信号を、一般に**雑音** (noise)と呼ぶ。通信の目的とするところは、信号電力の伝送ではなくて信号雑音比、すなわち**SN比**(signal-to-noise ratio)の十分大きな伝送を行うところにある。

雑音としては、電子の熱運動に起因して発生する**熱雑音**が有名であり、これは電子回路の抵抗素子などに必ず発生し、絶対温度に比例した電力を有する。あらゆる周波数成分を均一に含み、レベルの分布がガウス分布に従うことから、**白色ガウス雑音**(white Gaussian noise)と呼ばれる。また、すべての周波数成分を均一に含まない場合は、**有色雑音**(colored noise)と呼ばれる。

通信の途中に、別の通信に使われている信号が洩れてきて邪魔をすることがある。通信にとって有害な希望信号以外の信号を**干渉信号**と呼び、いろいろな原因が考えられる。

移動通信・携帯通信では2.2.2のように、ある一定の距離以上離れたセルでは、同じ時間に同じ周波数の電波を再利用して、電波の利用効率を上げる工夫がなされている。
このような通信システムでは電波の多重経路伝播により、無線で受信される信号の強さや位相が、時間・場所・周波数によって変動するフェージングという現象が生じSN比の減少につながる。一般的には通信品質を決定するのは**同一チャネル干渉**(co-channnel interference)と呼ばれる干渉信号との比である**CI比**(carrier-to-interference ratio)がSN比の代わりにその役割を担う。
同一チャネル干渉とは、同じ周波数チャネルを使っている別の送信信号が、目的の受信信号に重なって妨害になる現象であり、CI比は欲しい信号(C)と妨害信号(I)の強さの比を表す。
さらに、高速信号伝送時には、電波の多重経路伝搬による**符号間干渉**(intersymbol interference)も大きな問題となる

電話においては、多数のより対線が並行して束ねられて伝送されている区間がある。この場合、伝送路間に電磁的な結合が生じ、片方の伝送信号が他方の線路に漏れる。これを、**漏話**（cross talk）と呼ぶ。送信端側へ戻ってくる近端漏話の場合は雑音となるが、受信端側へ漏れる遠端漏話の場合には位相がそろうため、音声が漏れて聞こえる。これを防ぐためには、電磁的な結合を避ける必要があり、並行している線路をよったり、位置を入れ換えたりして、平均的に電磁結合を相殺するような工夫がなされる。

また、電話の場合、加入者の近くでは2線式伝送が行われているが、中継回線などでは、増幅器が入る関係で4線式伝送が行われる。そのため、途中で2線−4線変換が行われるが、その変換回路で信号の一部が漏れ、送話者側へ戻ってくる。とくに静止衛星を通した場合や、音声コーデック（codec）を通して遅延時間が耳に認識されるようになると、耳障りな**エコー**（反響）となる。これに対しては、最近は擬似エコーを生成してエコーを相殺する**エコーキャンセラー**（echo canceller）回路が開発され、解決されつつある。

信号の中継器などでは増幅器が使用されるが、非線形な入出力特性を持っていると、出力信号に入力信号周波数の和や差の周波数が新たに生成される。この周波数が受信信号の中に入ってくると、ひずみとして認識される。とくに周波数分割多重伝送などでは、このひずみが大きな影響を与えることになる。このひずみは、**非線形ひずみ**（nonlinear distortion）と呼ばれる。

以上、通信を妨げるいくつかの要因について説明した。これら以外にも伝送路の周波数特性が一様でないために発生する線形ひずみなど、いろいろな要因が考えられる。

## 2.3 情報の符号化
任意のアナログ信号は、後述の、標本化、量子化、符号化という操作により、ぢ時たる信号に変換可能である。ディジタル信号は雑音に強く、IC化されたディジタル信号処理回路により各種の処理が可能となり、長距離にわたって信頼度が高い信号伝送が行える特徴を有する。ここではディジタル信号の符号化について簡単に考察する。
### 2.3.1 数字の符号化
1. 自然2進符号
通常のディジタル信号伝送では、2進数が多用される。多値伝送の場合でも、2進数の組み合わせとして表現される場合が多い。任意の整数Nは以下のように表現が可能となる。
$$N = a_{n-1} \cdot 2^{n-1} + a_{n-2} \cdot 2^{n-2} + \cdots + a_1 \cdot 2 + a_0 $$
その係数$a_{n-1},\ a_{n-2},\ \cdots,\ a_2,\ a_1,\ a_0$が2進表現となる。逆に2進表現が与えられると上記ステップを逆にたどることにより安易に元の整数が計算可能である。ただし、2進表現では桁数が多くなるため、3桁ずつ区切った8進表現や、4桁ずつ区切った16進表現もよく用いられる。4桁の自然2進符号を、次に説明する反射2進符号と比較して表2.1に示す。


2. 反射2進符号(グレイ符号)

通常の自然2進符号を$a_{n-1},\ a_{n-2},\ \cdots,\ a_2,\ a_1,\ a_0$とすると
反射 2 進符号 $b_{n-1}, b_{n-2}, \dots, b_0$ は次のように計算される（$a_0, b_0$ を最下位桁とする）． $$b_{n-1} = a_{n-1}$$ $$b_i = a_{i+1} \oplus a_i \quad (i = n-2,\ n-3,\ \dots,\ 0) \tag{2.2}$$ ただし，$\oplus$ は mod2 加算である．

モデム（modem）でよく使われるディジタル多相位相変調信号では，搬送周波数の位相が情報を担う．8 相位相変調（8-PSK）では，**図 2・2** に示すように $\pi/4$ ごとの 8 つの位相が，それぞれ 3 ビットの情報を担う． このような変調方式の場合，もっとも誤りやすいのは隣接位相である．そこで，各位相に対する 2 進ビットの割当てを，反射 2 進符号にしたがって割り当てると，雑音によるビット誤り率が小さくできる．

### 2.3.2 文字の符号化
蓄積したデータやプログラムをお互いに交換するためには，共通の標準化された文字符号を定める必要がある．英文を対象とした情報交換用の符号としては，**ASCII**（American Standard Code for Information Interchange：アスキーと読む）符号や **EBCDIC**（Extended Binary Coded Decimal Interchange Code）が有名である．
